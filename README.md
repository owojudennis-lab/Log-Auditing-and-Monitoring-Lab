# Log Auditing & Monitoring Lab

## Objective
The Log Auditing & Monitoring Lab focused on tracking and analyzing system and file activity across both Linux and Windows environments. The goal was to understand how system logs capture user actions such as file access, deletion, and modifications, and how these logs can be used for security monitoring and incident investigation.

---

### Skills Learned
- File and system activity auditing
- Log analysis using Linux and Windows tools
- Detection of unauthorized or suspicious actions
- Understanding audit policies and event tracking
- Correlating system events for investigation

---

### Tools Used
- Ubuntu Linux
- Windows Event Viewer
- Linux Log Files (`/var/log/`)
- Audit Policies (Windows)
- File System Monitoring

---

## Steps

*Ref 1: File Deletion Audit*  
Captured logs showing deletion of a monitored folder.

*Ref 2: File Access Monitoring*  
Tracked file handling and modification activities.

*Ref 3: Linux Log Tracking*  
Analyzed logs on Ubuntu to monitor system activity.

*Ref 4: Event Viewer Analysis*  
Used Windows Event Viewer to review security events and audit logs.

---

## Key Outcome
Successfully demonstrated the ability to monitor, analyze, and interpret logs from both Linux and Windows systems to detect suspicious activity.
